1 Change DNS at GoDaddy

GoDaddy → your domain → Manage DNS. Make exactly these changes.

Apex (@) — replace the two old A records

ActionTypeNameValue
DeleteA@3.33.130.190
DeleteA@15.197.148.33
AddA@185.199.108.153
AddA@185.199.109.153
AddA@185.199.110.153
AddA@185.199.111.153

www — edit the existing CNAME

ActionTypeNameValue
EditCNAMEwwwcorbettclaims.net → mcorbett51090.github.io

Optional — IPv6 (AAAA on @)

2606:50c0:8000::153, 2606:50c0:8001::153, 2606:50c0:8002::153, 2606:50c0:8003::153

⚠ Do not touch these

  • Leave every MX and TXT record alone — those run assignments@corbettclaims.net email. Repointing the web A/CNAME records does not affect email; deleting an MX/TXT would break it.
  • If GoDaddy domain Forwarding is on for this domain, turn it off — it can override the A records.

2 Attach the domain

After DNS is in:

  1. Merge the domain-cutover PR (adds the CNAME file = corbettclaims.net and makes the site's URLs host-agnostic) — PR #3.
  2. Repo → Settings → Pages should show corbettclaims.net verified.
  3. Once the certificate finishes provisioning (minutes, occasionally up to an hour), tick Enforce HTTPS.

Merging before DNS is switched is harmless: the github.io URL just starts redirecting to corbettclaims.net, which still serves the old host until Step 1 completes.

Verifying the cutover

  • ✓DNS propagation: dig +short corbettclaims.net A should return the four 185.199.10x.153 addresses.
  • ✓Pages status: gh api repos/mcorbett51090/corbett-claims/pages → cname reads corbettclaims.net, https_enforced: true.
  • ✓End to end: load https://corbettclaims.net/ and submit the contact form — it should open a pre-filled email to assignments@corbettclaims.net.

Rollback

If anything goes wrong, restore the two original apex A records at GoDaddy (3.33.130.190, 15.197.148.33) and set the www CNAME back to corbettclaims.net. Because the TTL is 600s, the old host is reachable again within about 10 minutes. Keep a note of these originals before you change anything.
